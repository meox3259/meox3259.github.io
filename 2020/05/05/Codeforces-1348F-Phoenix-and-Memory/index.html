Codeforces 1348F - Phoenix and Memory

Solution:

搞了一个能解决这个问题一半的做法。

首先假设这个序列能被唯一还原，那么这个序列肯定存在某些性质，从正向思考，肯定存在一个长度为$1$的区间，这个区间只有一个选择，删除这个区间之后，又会生成这样的区间，直到结束。但是这样并不能维护，于是倒着考虑，发现一个区间的答案肯定是这个区间中被覆盖次数最少的数，并且覆盖次数为$1$。这个东西比较好维护，搞一棵线段树，支持区间加减，全局最小值，由于每次删除一个数，这个位置的最小值变成$0$，需要把这个位置变成正无穷。然后有一个问题，需要查找那个唯一覆盖这个点的区间，于是在线段树上每个节点开一个$set$，维护覆盖这个区间的线段集合，再支持一个区间加入区间删除即可。

以上这个做法能判断$YES$并输出方案，但是由于利用了一些特性，不能推广到一般的情况，于是需要其他的方法。

看见区间问题，自然想到扫描线，于是搞一个扫描线，那么考虑扫描到位置$i$，也就是数$i$，考虑应该把这个数填到哪个位置。用一个$set$维护位置对应的区间，那么自然，填右端点最小的区间最优，于是重复这个过程就可以构造出解，然后考虑两组解，交换两个位置，注意这里两个位置不一定相邻，因为会存在长度为$1$的区间隔开，其实在扫描的过程中，记录下第二靠左的区间，考虑是否能交换，这样是最优的，枚举一下即可。

时间复杂度$O(nlog^2n)$。需要卡常数，第二个做法是$O(nlogn)$的，很快。所以看见区间题，不要想，直接扫描线就行了。
