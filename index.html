ABC143F - Distinct Numbers

Posted on 2020-03-24 In Competitive Programming

Link:

ABC143F - Distinct Numbers

Solution:

先开始发现先削大的肯定最优，但是没办法维护区间减、区间第$k$大。然后又想计算多少个多出来的，发现如果一个堆大于$\frac{n}{k}$会使答案减少，其实这里已经接近答案了，不过这个方法不太对，好像会受到其他格子影响。还想到了二分判定，这也很接近答案了，可惜判定的方法不对，想把数字平均分堆，这是做不到的。

考虑正接，假设当前为$k$时，答案为$x$是否合法。发现所有数量大于$x$的堆，只能贡献$x$，小于等于$x$的堆，则全部可以贡献，这里是一个类似$Hall$定理的方法，那么由于$k$增大时答案递减，线性维护即可。

时间复杂度$O(n)$,不考虑排序的复杂度。

这种想法是计算每个堆的贡献，先开始想到的计算每个堆扣多少，只有一步之遥。

ABC145F - Laminate

Posted on 2020-03-17 Edited on 2020-04-01 In Competitive Programming

Link:

ABC145F - Laminate

Solution:

想了很长时间，其实挺简单。先开始看见数据范围想区间$dp$，发现不太会，想了笛卡尔树，发现不太行。

先思考如何计算答案，答案$= \sum_{i = 1}^{n}{max(0,a_{i} - a_{i - 1})}$。

那么答案只与相邻两项有关，问题简化了许多。然后考虑一个$dp$，$dp_{i,j,k}$表示前$i$个数修改了$j$次，最后一个值是$k$。当然需要将序列里的值离散化一下。然后转移的时候维护一个前缀最小值和后缀最小值即可。

时间复杂度$O(n^3)$。

这种选几次的问题，如果不能贪心，就只能有背包$dp$，而且不能被高维状态吓到，可以优化转移。

ARC064B - An Ordinary Game

Posted on 2020-03-16 Edited on 2020-03-19 In Competitive Programming

Link:

ARC064B - An Ordinary Game

Solution:

想了挺长时间，先开始没看见相邻不相等。

假设所有字母不一样，那么长度是计数第一个人赢，否则第二个人。考虑有相同的字母，如果首尾不同，考虑几个不可以动，一定是成对不可以动，例如$ABAB$，所以仍然是奇偶性。再考虑首尾相同，如果有不可以动的，也是成对出现，例如$ABBA$。

感觉相邻不同是防止了出现连续一段导致很麻烦的情况。

时间复杂度$O(n)$

Codeforces 1325F - Ehab's Last Theorem

Posted on 2020-03-16 In Competitive Programming

Link:

Codeforces 1325F - Ehab’s Last Theorem

Solution:

选中一个点开始$dfs$，构成一颗$dfs$树。如果存在一个点有$\geq \lceil \sqrt{n} \rceil$条反祖边，可以得出存在一个满足条件的环。

如果不存在满足的环，说明每个点的反祖边$\leq \lceil \sqrt{n} \rceil$。考虑条件$2$，每次离开一个点时，如果这个点没有被标记，那么加入答案，标记反祖边对应的点，这样至少会加入$\lceil \sqrt{n} \rceil$个点，满足条件，因为不用考虑非反祖边，这些边之前已被删除。

时间复杂度$O(n)$

先开始想了一个跟点双有关的算法，但是细节不是很完善，并且不能找出环。然后又想关于最大团的方法。这时候思维就局限了，应该考虑一些其他的方法，如度数相关，$dfs$树之类的。

ABC147F - Sum Difference

Posted on 2020-03-12 Edited on 2020-03-16 In Competitive Programming

Link:

ABC147F - Sum Difference

Solution:

首先可以转换一下，$S - T = 2 * sum - S$那么问题转化成问有多少种不同的$S$。

最先开始一直想枚举$X$的个数，然后计算通过选$A$凑出来$X$的倍数，不过这样涉及容斥一类，很麻烦。

发现任意的$A$可以表示成$KX + DY$形式，其中$K$，$D$均为整数。看到这个样子就能想到两个数相等的基本条件就是模$D$相等，也可以通过数形结合发现。按余数分类，发现对于每个$K$，所组成的$A$除以$D$是连续的一段区间，这样问题转化成了线段的并，扫描线即可。

时间复杂度$O(nlogn)$

看见等差数列一类的可以考虑数形结合和余数，这样的套路出现过很多次，这样就可以把相等的条件通过分类简化，使得计算相等容易,就像$cf$的那道按奇偶性分点一样。

AGC035B - Even Degrees

Posted on 2020-03-12 Edited on 2020-03-16 In Competitive Programming

Link:

AGC035B - Even Degrees

Solutions:

题目要求每个点出度为偶数，先将所有边随意定向，这样存在一些奇数点和一些偶数点。发现改变一条边会使端点奇偶性改变，这样奇数点的数量和偶数点的数量的奇偶性不会改变，所以如果存在奇数个奇数点则不合法。

考虑修改，每次找出两个奇数点，翻转他们之间的一条路径即可。于是先找出一颗$dfs$树,每次找出两个奇数点，把之间路径翻转即可，这里用树上差分。

时间复杂度$O(nlogn)$
